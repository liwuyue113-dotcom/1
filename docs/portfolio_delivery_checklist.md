# 作品集交付清单最小整理

## 文档用途

本文记录 2026-06-24 当前已经能展示的作品集模块、推荐录屏顺序和仍需补的高优先级缺口。目标是帮助后续进入“交付收束”，而不是继续打磨已经达到最低可用度的局部功能。

本阶段只做交付清单，不新增代码功能，不修改 Unity 场景。

## 推荐录屏顺序

第一组：主作品演示

1. 《囚城营救》Level1 基础游玩：展示倒计时、移动、敌人和三路线选择。
2. 胆小守卫真路线：安抚守卫，获得“往前第二个口子跳下去”的真实情报，并找到下方暗道入口。
3. 胆小守卫假路线：威胁守卫，获得“往上走最安全”的误导情报。

第二组：AI 策划能力演示

4. RAG 世界观问答：询问真实路线或角色设定，展示回答和来源。
5. RAG 冲突检查器：输入与暗道守卫数量冲突的候选设定，展示工具提醒可能冲突。
6. AI 关卡设计助手：生成关卡方案并输出评价结果。

第三组：Agent 技术扩展演示

7. Agent Workflow 命令行：展示白名单工具、记忆记录和未授权工具拒绝。
8. MCP 命令行演示：展示 `tools/list`、`get_route_info` 和 `check_game_state`。

## 高优先级缺口

| 缺口 | 为什么优先 | 最小下一步 |
| --- | --- | --- |
| 作品集入口说明还不够集中 | 面试官或老师需要先看到项目整体价值 | 新增或整理一个 README / 作品集总览入口 |
| Git 仓库尚无正式提交 | 当前文件很多未跟踪，后续容易丢失或混乱 | 做一次提交前清理和 `.gitignore` 检查 |
| 外部 FastAPI 项目交付关系 | 已完成最小说明 | 按 `docs/external_fastapi_delivery.md` 启动和验证；当前不迁移代码 |
| Unity 实际项目不在当前仓库内 | 代码、场景和文档分离，交付时容易讲不清 | 在 README 中明确 Unity 项目路径和当前仓库职责 |
| 正式录屏素材命名 | 已完成最小整理 | 按 `portfolio_delivery/README.md` 中的 `01` 到 `08` 命名录制 |
