# AI 守卫路线提示录屏最小复核

## 文档用途

本文用于 2026-06-24 之后的下一次录屏前复核。目标是确认“玩家通过胆小守卫获得真实情报，然后找到下方暗道入口”这条展示流程是否连贯。

本阶段只做录屏脚本和复核清单，不改 Unity 场景，不改 AI NPC 逻辑，不新增剧情。

## 本阶段目标

最低可用展示链路：

```text
玩家靠近胆小守卫
-> 输入安抚和询问路线
-> 获得真实情报
-> 屏幕出现路线提示
-> 玩家离开守卫
-> 根据“往前第二个口子跳下去”找到下方暗道入口
```

只要这条链路可以被录成一段 60 到 90 秒的视频，本阶段就达到最低可用度。

## 录屏前置条件

- FastAPI 服务已启动。
- Level1 可以正常进入。
- 玩家靠近胆小守卫时，AI 对话 UI 会显示。
- 玩家远离胆小守卫时，AI 对话 UI 会隐藏。
- 倒计时、路线提示 UI 和玩家移动都正常显示。
- 不显示 API Key、环境变量或无关终端窗口。

## 最小录屏步骤

1. 从胆小守卫附近但 UI 未出现的位置开始录制。
2. 走近胆小守卫，展示 AI 对话 UI 出现。
3. 输入安抚句子：

```text
别怕，我不会伤害你，我只是想找到更安全的路。
```

4. 展示守卫回复、情绪变化和首次对话加时。
5. 继续安抚或直接询问路线：

```text
哪条路线的守卫最少？
```

6. 展示真实路线提示：

```text
往前第二个口子跳下去有一条暗道。
```

7. 玩家离开胆小守卫，展示 AI 对话 UI 隐藏。
8. 沿着提示方向前进，走到下方暗道入口。
9. 镜头停在入口附近，证明提示能帮助玩家找到目标。

## 成功判定

| 检查项 | 通过标准 |
| --- | --- |
| 对话触发 | 靠近胆小守卫后 AI 对话 UI 出现 |
| 情报获得 | 玩家能通过安抚和询问路线得到真实情报 |
| 路线提示 | 屏幕能看到“往前第二个口子跳下去”的暗道提示 |
| UI 收起 | 玩家离开守卫后 AI 对话 UI 隐藏 |
| 入口可找 | 玩家能根据提示找到下方暗道入口 |
| 录屏清晰 | 倒计时、玩家、守卫、输入框、回复和路线提示都能看清 |

## 实际试跑结论

2026-06-24，用户按本清单试跑真路线流程，结论为：可以从胆小守卫真实情报顺利走到下方暗道入口。

因此，本阶段达到最低可用度。当前不需要继续扩写录屏脚本，也不需要调整 Unity 场景。

## 如果不通过

只记录具体失败点，不在本阶段扩大修改范围：

- 如果 UI 不出现，回到 AI 交互范围检查。
- 如果没有真实情报，回到对话规则或输入句子检查。
- 如果路线提示不显示，回到 Unity 路线提示 UI 检查。
- 如果玩家找不到入口，回到暗道入口提示表现检查。

## 本阶段不做

- 不录威胁获得假路线流程。
- 不新增路线。
- 不重摆敌人。
- 不调整战斗数值。
- 不接 RAG 或 MCP 到 Unity。
- 不重写已有 `docs/ai_npc.md` 录制指南。
