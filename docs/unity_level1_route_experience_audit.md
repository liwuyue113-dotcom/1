# Unity Level1 路线整体体验最小核对

## 文档用途

本文记录 2026-06-24 对 `Level1.unity` 三条路线体验差异的最小核对。目标不是重新设计关卡，也不是修改 Unity 场景，而是确认下一次进入 Unity 编辑器时应该优先看哪些体验问题。

本阶段只做静态坐标整理和复核清单，不直接手改 `.unity` YAML。

## 本阶段目标

三条路线的最低可用体验目标来自 `docs/game_design.md`：

| 路线 | 体验目标 |
| --- | --- |
| 上路 | 路线更长或更绕，敌人压力一般 |
| 中路 | 路线长度一般，敌人压力更集中 |
| 下方暗道 | 入口默认不明显，获得胆小守卫真实情报后能理解入口方向，敌人最少 |

## 当前可确认结论

- Level1 已经具备三条路线所需的高度层级。
- 上方层级有多个敌人点位，具备形成路线压力的基础。
- 中部层级有一组较集中的敌人点位，具备表现中路压力的基础。
- 下方暗道的 3 个陷阱敌人不计入可走路径上的站岗守卫数量。
- 胆小守卫和路线提示 UI 已存在，可以承接“获得真实情报后理解暗道入口”的展示目标。

## 仍不能只靠静态文件确认的内容

以下内容必须在 Unity Scene 视图或实际游玩中确认：

| 问题 | 为什么不能只靠静态坐标判断 |
| --- | --- |
| 上路是否真的更长或更绕 | 坐标只能说明对象位置，不能说明玩家实际走法、墙体阻挡和路径长度 |
| 中路是否比上路压力更集中 | 坐标不能判断敌人视野、巡逻范围、玩家必经路径和战斗触发概率 |
| 下方暗道入口是否默认不明显 | 入口可见性依赖瓦片、美术遮挡、镜头和玩家视角，不能只看 YAML |

## Unity 实测复核结论

2026-06-24，用户在 Unity 编辑器或实际游玩中确认：

| 复核问题 | 结论 |
| --- | --- |
| 上路是不是明显更绕或更耗时 | 是 |
| 中路是不是更容易看到密集守卫或形成集中压力 | 是 |
| 下方暗道入口是不是默认不明显，但听到“往前第二个口子跳下去”后能找到 | 是 |

因此，当前 Level1 的三条路线已经达到最低可用的体验差异：

- 上路体现“更绕 / 更耗时”。
- 中路体现“守卫更密 / 压力更集中”。
- 下方暗道体现“入口不明显，但真实情报能帮助玩家理解入口方向”。

本阶段不需要继续打磨路线长度、敌人数量或入口表现。

## 下一阶段最小复核清单

下一次进入 Unity 编辑器或实际游玩时，只确认三个问题：

```text
1. 上路是不是明显更绕或更耗时？

3. 下方暗道入口是不是默认不明显，但听到“往前第二个口子跳下去”后能找到？
```

这三个问题已经完成复核，本阶段达到最低可用度。不要在这一步新增路线、重摆全部敌人或调整战斗数值。
