# 《囚城营救》AI NPC 阶段式项目路线图

更新日期：2026-06-08

## 使用规则

- 每次只推进一个阶段。
- 每个阶段只完成最低可用版本。
- 用户反馈运行结果后，先进行阶段验收。
- 没有阻塞下一阶段的问题时，立即进入下一阶段。
- 一旦某阶段达到最小可行性，必须停止继续补充、打磨或扩写该阶段内容，并开始下一阶段。
- 高级优化、扩展架构和备选玩法放入后续阶段，不提前实现。

每次教学回复必须包含：

```text
当前阶段：
本阶段目标：
是否完成：
下一步要做什么：
本次只需要修改的文件：
```

## 路线总览

```text

-> 第 2 阶段：DeepSeek 判断玩家意图

```

## 第 1 阶段：Python 规则原型

目标：不依赖 AI 和 Unity，让胆小守卫的基础玩法规则在 Python 控制台中运行。

最低可用功能：

- 玩家可以连续输入文字。
- 安抚会增加信任、降低恐惧。
- 威胁会降低信任、增加恐惧。
- 询问路线时，系统能返回 `true_route`、`false_route` 或 `none`。
- 对话历史和状态能保存到 JSON。

验收标准：

- 程序能够运行并正常退出。
- 安抚、威胁、问路三种输入能得到符合规则的结果。
- 重新运行后能读取之前保存的状态。
- 不存在阻塞 DeepSeek 接入的运行错误。

当前状态：已于 2026-06-09 完成验收。

本阶段不做：

- 继续扩展关键词。
- 高级意图分类。
- FastAPI、Unity 或 RAG。

## 第 2 阶段：DeepSeek 判断玩家意图

目标：用 DeepSeek 替代关键词判断，让模型只判断玩家意图。

最低可用功能：

- Python 能调用 DeepSeek API。
- 输入玩家话语后，DeepSeek 返回一个意图：
  - `comfort`
  - `threaten`
  - `ask_route`
  - `other`
- 现有 Python 规则根据意图更新信任、恐惧和情报结果。

验收标准：

- “放心，我不会伤害你”能被识别为 `comfort`。
- “不说我就杀了你”能被识别为 `threaten`。
- “告诉我安全路线”能被识别为 `ask_route`。
- API 失败时程序不会崩溃。

本阶段不做：

- 让 AI 决定信任数值。
- 让 AI 决定真假路线。
- 生成复杂角色台词。

当前状态：已于 2026-06-09 完成验收。DeepSeek 已正确识别 `comfort`、`threaten`、`ask_route`、`other`，并且现有 Python 规则继续负责状态变化。

## 第 3 阶段：DeepSeek 生成守卫台词

目标：让守卫根据已确定的状态和结果生成符合角色设定的回复。

最低可用功能：

- Python 将守卫状态、玩家输入和规则结果发送给 DeepSeek。
- DeepSeek 返回守卫台词。
- 真路线、假路线和拒绝透露时，台词符合对应结果。

验收标准：

- 台词符合胆小守卫性格。
- AI 不会在 `none` 状态下擅自透露路线。
- AI 台词异常时使用预设回复。

本阶段不做：

- RAG。
- 长期记忆摘要。
- 多 NPC。

当前状态：已于 2026-06-09 完成验收。已验证 `true_route`、`false_route` 和 `none` 三种结果；`none` 状态下的路线泄露问题已通过限制 Prompt 信息和 Python 输出拦截解决。

## 第 4 阶段：FastAPI 对话接口

目标：把 Python AI NPC 变成 Unity 可以调用的本地服务。

最低可用功能：

- `GET /health` 返回服务正常。
- `POST /npc/chat` 接收玩家输入。
- 接口返回固定 JSON：

```json
{
  "reply": "守卫回复",
  "emotion": "非常害怕",
  "trust": 10,
  "fear": 70,
  "intel_result": "false_route"
}
```

验收标准：

- 浏览器或接口测试工具能够访问健康检查。
- 对话接口能返回正确 JSON。
- API 请求失败不会破坏守卫状态。

当前状态：已于 2026-06-09 完成验收。`GET /health` 与 `POST /npc/chat` 均已跑通；对话接口返回 HTTP `200` 和完整结构化 JSON。

子目标进度：

- [已完成] FastAPI 与 Uvicorn 安装
- [已完成] FastAPI 服务启动与 `GET /health`
- [已完成] `POST /npc/chat`

## 第 5 阶段：Unity 调用 FastAPI

目标：Unity 能发送一句玩家输入，并收到 Python 返回结果。

最低可用功能：

- Unity 使用 HTTP POST 调用 `/npc/chat`。
- Unity Console 能打印守卫回复和情报结果。

验收标准：

- Unity 能成功连接本地 FastAPI。
- 中文输入和回复正常。
- 请求失败时 Unity 不会卡死。

当前状态：已于 2026-06-09 完成验收。Unity 已成功向 `/npc/chat` 发送固定中文消息，并在 Console 打印 FastAPI 返回的中文结构化 JSON；请求使用协程执行，不阻塞 Unity 主循环。

## 第 6 阶段：Unity AI 对话 UI

目标：玩家能够在游戏内输入文字并看到守卫回复。

最低可用功能：

- 输入框。
- 发送按钮。
- 守卫回复文字。
- 守卫情绪文字。
- 请求期间避免重复发送。

验收标准：

- 玩家能在 Level1 与胆小守卫连续对话。
- 对话期间倒计时继续运行。
- API 失败时显示预设回复。

本阶段不做：

- 精美 UI 动画。
- 语音输入。
- 复杂聊天记录滚动效果。

当前状态：已于 2026-06-10 完成验收。玩家能够输入中文并连续与守卫对话；界面显示守卫回复和情绪；请求期间发送按钮与输入框被禁用，请求结束后恢复；倒计时在对话期间继续运行。当前单次响应约需 4 到 5 秒，但不会阻塞 Unity，暂不在本阶段优化。

## 第 7 阶段：真假路线影响游戏机制

目标：AI 对话结果真正影响限时营救玩法。

最低可用功能：

- 首次对话只增加一次 20 秒。
- `true_route` 显示真实安全路线文字。
- `false_route` 显示虚假危险路线文字。

验收标准：

- 真情报和假情报会产生不同游戏结果。
- 假情报不会额外增强敌人。
- 同一局中情报可以从假路线更正为真路线。

当前验收进展：

- 已验证首次成功对话后触发增加 20 秒。
- 已验证后续对话不会重复增加 20 秒。
- 已验证 `true_route` 显示往前第二个口子跳下去的暗道提示，`false_route` 显示往上走的虚假提示。
- 已验证同一局中假路线能够更正为真路线。

后期再做：

- `true_route` 降低安全路线指定守卫血量。当前尚未确定目标守卫和现有血量脚本接口，不阻塞本阶段。

当前状态：已于 2026-06-10 完成验收。首次成功对话只增加一次 20 秒；假情报提示往上走最安全；真情报提示往前第二个口子跳下去存在几乎无人镇守的暗道；假路线能够在后续成功安抚后更正为真路线。

## 第 8 阶段：Level1 完整整合

目标：把胆小守卫作为 Level1 中可完整体验的玩法节点。

最低可用功能：

- NPC 位置、触发范围和剧情理由合理。
- 玩家能通过安抚或威胁获得不同结果。
- 路线提示能够帮助或误导玩家。
- 重开关卡后状态按设计重置。

验收标准：

- 从进入 Level1 到获得路线情报的流程能完整跑通。
- 不存在阻塞通关的严重 bug。
- 普通剧情 NPC 不受 AI 守卫功能影响。

当前验收进展：

- 已验证 AI 守卫对话 UI、倒计时加时和真假路线提示已在 Level1 中运行。
- 已验证玩家能够通过安抚或威胁获得不同路线结果。
- 已验证重开关卡后 Unity 与 Python 守卫状态按设计重置。
- 已验证普通剧情 NPC 不受 AI 守卫功能影响。
- 已验证玩家远离胆小守卫时 AI 对话 UI 隐藏，靠近时正常显示。

重置验证结果：

- 已通过实际重开关卡测试确认 Unity 与 Python 守卫状态按当前设计正常重置。

当前状态：已于 2026-06-12 完成验收。从进入 Level1 到获得真假路线情报的流程能够完整跑通；重开关卡状态正常重置；普通剧情 NPC 不受影响；AI 对话 UI 只在玩家靠近胆小守卫时显示。

## 第 9 阶段：测试、展示与作品集整理

目标：形成可录制、可讲解、可投递的作品集内容。

最低可用功能：

- 录制安抚获得真路线的流程。
- 录制威胁获得假路线的流程。
- 绘制简单架构图。
- 整理项目介绍、技术职责和策划价值。

验收标准：

- 演示过程稳定。
- 能解释 AI、Python 规则系统和 Unity 各自负责什么。
- 能说明 AI NPC 如何影响实际游戏玩法。

当前状态：已于 2026-06-12 完成。真路线和假路线演示流程能够稳定运行，录制指南、系统架构说明和作品集项目介绍已整理完成。

已知瑕疵：

- 退出并重新进入 Unity Play 模式后，Python 侧可能沿用上一局 `guard_state.json` 中的信任、恐惧、情绪和历史记录。后续需要在新一局开始时重置 Python 守卫状态。
