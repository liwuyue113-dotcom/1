## 项目总体介绍

作品集以 Unity 2D 游戏《囚城营救》为核心，逐步结合 Python、Prompt、API、FastAPI、AI NPC、RAG 游戏设定知识库、AI 关卡设计助手、MCP / Tool Calling / Memory、LangGraph 和多 Agent 游戏世界。

《囚城营救》是一款基于《九州缥缈录》原著小说人物与世界观创作的原创同人支线 2D 潜入营救游戏。玩家扮演姬野，需要潜入离国边境城堡、躲避巡逻、寻找钥匙，并在黎明前找到被离国军俘获的昔日战友吕归尘。

本项目原创内容是潜入营救支线、关卡流程与玩法实现。原著人物、势力与人物关系归原作者及相关权利方所有。

## 已确认的 AI NPC：胆小守卫

胆小守卫听见外面的砍杀声后擅自离开岗位，躲在 Level1 深处。玩家遇见他时，他非常害怕，并认为玩家可能是杀人魔。

核心策划目标：AI 对话必须影响实际玩法，而不是只提供装饰性聊天。

```text
玩家语言
-> 改变守卫的信任和恐惧
-> 决定守卫提供真情报、假情报或拒绝透露
-> 影响玩家在限时营救中的路线选择
```

已确认决策：

- NPC 位于 `Level1`。
- 玩家使用自由文字输入，不使用固定对话选项。
- 守卫记住当前一局中的对话，重新开始关卡后记忆清空。
- 玩家看不到具体信任和恐惧数值，只能看到情绪与语气。
- 安抚守卫可以稳定获得真实安全路线。
- 普通威胁可能让守卫为了自保而提供假路线。
- 玩家可以质疑假情报并继续安抚，让守卫改口。
- 玩家通过措辞与情绪判断情报真假，系统不直接标注真假。
- 对话期间倒计时继续运行。
- 玩家首次触发该对话时只获得一次 `+20 秒`。
- 真实路线由策划预先设定，AI 不能编造地图路线。
- 真实情报指向往前第二个口子跳下去的暗道，几乎无人镇守。
- 假情报误导玩家往上走，该路线较危险，但不额外增强敌人。
- DeepSeek API 失败时使用预设台词，不改变状态，并允许玩家重试。
- 真实情报降低指定守卫血量属于后期扩展，当前尚未实现。

架构原则：DeepSeek 负责判断玩家意图和生成角色台词；Python / FastAPI 保存状态并按规则决定真假情报；Unity 执行倒计时、UI、路线提示和其他可信游戏效果。

## 技术路线

完整长期学习路线见根目录 `agent.md` 第 3 节。当前项目按“先完成可展示作品，再逐步深入技术”的原则推进。

### 阶段一：Python + Prompt + API + Unity 项目完成

- 巩固 Python 基础与文件读写
- 学习角色设定、关卡生成、剧情生成、NPC 行为和固定格式输出 Prompt
- 学习 `requests`、OpenAI SDK 或兼容 OpenAI 格式的 SDK
- 调用 DeepSeek / OpenAI 类大模型
- 继续完善《囚城营救》

### 阶段二：FastAPI + AI 关卡设计助手

- 学习 FastAPI 路由、请求和返回
- 前端调用后端，后端调用 AI
- 把 AI 输出变成固定策划模板
- 评价 AI 生成关卡质量
- 输出关卡主题、玩家目标、地图区域、安全路线、危险路线、隐藏路线、敌人、奖励、失败条件、设计意图和优化建议

### 阶段三：MCP + Tool Calling + Memory + Agent Workflow

- 理解 Agent 不是聊天机器人，而是会用工具、会记忆、会按流程完成任务的 AI
- 用游戏例子学习开门、查看地图、更新任务状态、写入玩家行为记忆
- 练习“目标 -> 规划 -> 行动 -> 反思 -> 继续行动”的工作流

### 阶段四：Embedding + 向量数据库 + RAG 游戏设定知识库

- 用“文字变成数字向量”的方式理解 Embedding
- 使用 Chroma 或 FAISS 等向量检索方案
- 建立角色、剧情、地图、任务设定知识库
- 支持来源引用、世界观问答、支线任务生成和设定冲突检查

### 阶段五：Unity + AI NPC

- Unity 通过 HTTP 请求 FastAPI
- FastAPI 调用 AI
- AI 根据 NPC 姓名、身份、性格、目标、秘密、状态、信任度和历史互动回答
- 对话影响隐藏路线、任务提示或其他游戏机制

### 阶段六：LangGraph + 多 Agent 游戏世界

- 学习多 Agent 状态流转、世界状态表、目标系统和行动系统
- 构建守卫、囚犯、盗贼、商人和玩家的小型城堡世界
- 限制 AI 只能在规则允许的行动中决策
