# 《囚城营救》游戏设计文档

## 文档用途

本文档记录《囚城营救》的核心策划设计。玩法、关卡或叙事发生重要变化时，应同步更新本文档，并在 `tech_decision.md` 中记录相关开发决策。

## 游戏定位

- 类型：Unity 2D 潜入营救游戏
- 同人定位：基于《九州缥缈录》原著小说人物与世界观创作的原创同人支线
- 核心体验：玩家需要在有限时间内权衡风险与效率，潜入城堡并营救目标
- 项目定位：可用于校招作品集展示的完整游戏项目
- 目标岗位：游戏策划、AI 游戏策划、剧情叙事策划
- 差异化方向：将传统潜入关卡与 AI NPC、RAG 世界观知识库、Agent 行为系统结合

## 故事背景

吕归尘在战争中被离国军俘获，被关押于离国边境城堡并接受审问。离国军将在黎明前把吕归尘转移至主力军营。

玩家扮演吕归尘的昔日战友姬野，独自潜入离国边境城堡。玩家需要躲避或击败巡逻守卫、寻找钥匙，并在押送队出发前找到吕归尘。

本支线为项目原创事件，不声称是《九州缥缈录》原著情节。原著人物、势力与人物关系归原作者及相关权利方所有。

## 核心玩法

核心玩法循环：

1. 观察地图、守卫与环境信息
2. 选择安全路线、危险路线或隐藏路线
3. 潜入、躲避巡逻，必要时进行战斗
4. 寻找钥匙或从 NPC 处获取路线情报
5. 在限时内营救目标
6. 根据当前局势规划逃脱

核心机制：

- 限时营救
- 守卫巡逻与潜入躲避
- 钥匙寻找
- 战斗
- 多路线选择
- AI NPC 对话与信任机制（计划加入）

## 关卡结构

地图包含三条主路：上路、中路和下方暗道路线。三条路线让玩家在路线长度、敌人数量和情报收益之间做选择：

| 路线 | 初步定位 | 优势 | 风险 | 当前状态 |
| --- | --- | --- | --- | --- |
| 上路 | 玩家可以直接选择的正常路线 | 敌人数量一般 | 路线较长，消耗更多时间 | 已存在于实际关卡 |
| 中路 | 玩家可以直接选择的正常路线 | 路线长度一般 | 敌人数量略多 | 已存在于实际关卡 |
| 下方暗道路线 | 通过胆小守卫真实情报发现的隐藏路线 | 长度正常，敌人最少 | 需要先与 AI NPC 对话并取得真实情报 | 已与胆小守卫 AI NPC 机制结合 |

胆小守卫当前路线情报设定：

- 真实下方暗道路线：往前第二个口子跳下去有一条暗道，几乎无人镇守。
- 虚假危险路线：守卫谎称往上走最安全。

当前三条路线的最低可用差异：

- 上路较长，敌人数量一般。
- 中路长度一般，敌人数量略多。
- 下方暗道路线长度正常，敌人最少，需要通过胆小守卫真实情报发现。

### 路线提示方式

| 路线 | 玩家提示方式 | 设计目的 |
| --- | --- | --- |
| 上路 | 入口看起来更远、更绕，可用较长走廊、拐弯或远处火把表现 | 让玩家理解上路更耗时，但敌人压力一般 |
| 中路 | 入口附近能看到更多守卫或更密集的巡逻点 | 让玩家理解中路长度一般，但敌人略多 |
| 下方暗道路线 | 默认不明显，需要胆小守卫真实情报提示“往前第二个口子跳下去” | 让玩家理解暗道敌人最少，但需要情报解锁 |

### 敌人站位

| 路线 | 敌人站位 | 设计目的 |
| --- | --- | --- |
| 上路 | 开始很分散，主要是一个一个的守卫；后面会有连续几个守卫站岗 | 前段压力较低，后段形成连续站岗压力 |
| 中路 | 敌人分布很多，有的地方有好几个守卫站岗 | 让玩家感到中路敌人略多，通行压力更集中 |
| 下方暗道路线 | 可走路径上只有一个站岗守卫；陷阱里的敌人不会出来，不计入站岗守卫数量 | 保持暗道敌人最少，同时保留最低风险点 |

### 巡逻方式

| 路线 | 巡逻方式 | 玩家压力 |
| --- | --- | --- |
| 上路 | 前半段守卫分散站岗或短距离巡逻；后半段连续几个守卫偏固定站岗 | 玩家前段可以逐个观察和处理，后段会遇到一段压力区 |
| 中路 | 守卫更密集，部分位置有多个守卫同时站岗 | 玩家更容易被迫战斗或等待空隙 |
| 下方暗道路线 | 暗道可走路径上只有一个站岗守卫，不做复杂巡逻；陷阱里的敌人不会出来 | 玩家主要挑战是找到暗道入口，而不是连续战斗 |

### Unity 场景核对清单

本清单用于进入实际 Unity 场景后逐项检查路线设定是否已经落到关卡里，不用于新增设定。

| 检查对象 | 核对项 |
| --- | --- |
| 上路 | 是否明显比中路和下方暗道更长 |
| 上路 | 前半段敌人是否分散，主要是一个一个的守卫 |
| 上路 | 后半段是否有连续几个守卫站岗，形成压力区 |
| 中路 | 路线长度是否处于一般水平 |
| 中路 | 敌人数量是否比上路略多 |
| 中路 | 是否存在多个守卫同时站岗的位置 |
| 下方暗道路线 | 是否需要胆小守卫真实情报才能明确入口方向 |
| 下方暗道路线 | 入口是否默认不明显 |
| 下方暗道路线 | 暗道可走路径上是否只有一个站岗守卫，陷阱里的敌人是否不会出来 |
| 提示方式 | 上路入口是否看起来更远、更绕 |
| 提示方式 | 中路入口是否能看出守卫更多或巡逻更密 |
| 提示方式 | 暗道是否通过“往前第二个口子跳下去”的情报提示 |

### Unity Level1 静态核对结果

核对日期：2026-06-17

最新核对：2026-06-24，详见 `docs/unity_level1_route_audit.md`。

路线体验最小核对：2026-06-24，详见 `docs/unity_level1_route_experience_audit.md`。

已确认事实：

- 实际 Unity 项目位于 `D:\UnityProjects\project03`。
- 主要核对场景为 `D:\UnityProjects\project03\Assets\scens\Level1.unity`。
- 当前 Unity 版本为 `2022.3.62f3`。
- Level1 中已存在 `AIguard`、`AIGuardDialogueSystem`、`AIGuardInteractionZone`、多个 `Enemy`、多个 `CheckPoint` 和多个 `sceneportal`。
- 静态坐标显示，场景中已有上、中、下多个高度层级的敌人分布。

Unity 编辑器人工复核结论：

- `Enemy (3)`、`Enemy (4)`、`Enemy (6)` 都位于暗道内部的陷阱里，不会出来。
- 这 3 个敌人不按“暗道可走路径上的站岗守卫”计算。
- 当前不需要移动、删除或新增敌人。

后续仍可进入 Unity 编辑器确认或调整：

- 上路需要确认是否明显更绕或更耗时。
- 中路需要确认是否更容易看到密集守卫或形成集中压力。
- 下方暗道入口需要确认是否默认不明显，但获得“往前第二个口子跳下去”的真实情报后能找到。

2026-06-24 Unity 实测复核结论：

- 上路已经能明显体现更绕或更耗时。
- 中路已经能看到密集守卫或形成集中压力。
- 下方暗道入口默认不明显，但获得“往前第二个口子跳下去”的真实情报后可以找到。
- 三条路线已经达到最低可用体验差异，当前不继续打磨路线长度、敌人数量或入口表现。

关卡设计需要持续验证：

- 三条路线是否具有清晰且不同的体验
- 时间限制是否能真正推动路线选择
- 战斗、潜入和探索是否都有价值
- 隐藏路线是否通过合理线索被玩家发现

## 美术风格

- 主题：中世纪幻想城堡

- 场景关键词：石墙、牢房、城堡走廊、守卫岗哨、隐秘密道

## 设计评价指标

- 玩家能否理解当前目标与剩余时间
- 不同路线是否带来有意义的选择
- 潜入与战斗操作是否有清晰反馈
- NPC 对话是否影响实际玩法，而不只是提供文本
- AI 功能是否增强策划设计，而不是喧宾夺主
