# Unity Level1 主演示录屏前最小检查

## 文档用途

本清单用于录制作品集第一条视频 `01_level1_overview.mp4`。目标是用 60 到 90 秒让观看者理解《囚城营救》的基础玩法和 Level1 路线选择。

本阶段只做录屏前检查，不修改 Unity 场景、脚本、敌人或数值。

## 本阶段作用

`01_level1_overview.mp4` 先回答“这是一个什么游戏”：

```text
进入 Level1
-> 看到倒计时和玩家角色
-> 展示移动与敌人压力
-> 让观看者理解存在上路、中路和下方暗道选择
-> 在路线选择或代表性路段处结束
```

胆小守卫对话、真路线情报和暗道入口闭环留给 `02_ai_guard_true_route.mp4`，避免两条视频重复。

## 已确认前置

2026-06-28 静态确认：

- Unity 版本为 `2022.3.62f3`。

## 录屏前检查

- 打开 `Level1` 并从新一局开始。
- 确认玩家移动、倒计时、敌人和主要 UI 正常。
- 确认 Game 视图画面完整，没有被 Inspector、Console 或其他窗口遮挡。
- 优先使用 `1920x1080` 横屏录制；如录屏软件不支持，以文字和玩家角色可看清为准。
- 关闭桌面通知，避免弹窗进入画面。
- 确认录制文件名为 `01_level1_overview.mp4`。

本条视频不需要启动 FastAPI 或 DeepSeek，因为 AI 对话属于后续独立演示。

## 最小录屏顺序

| 时间 | 展示内容 | 成功信号 |
| --- | --- | --- |
| 0-10 秒 | 从 Level1 正常游戏画面开始 | 玩家角色和倒计时清晰可见 |
| 10-30 秒 | 展示基础移动和潜入节奏 | 玩家操作连贯，不长时间停留 |
| 30-50 秒 | 展示至少一组敌人或巡逻压力 | 观看者能理解需要观察、绕行或战斗 |
| 50-75 秒 | 经过路线分流位置或代表性路段 | 观看者能理解关卡不只有一条直线通路 |
| 75-90 秒 | 在路线选择或清晰场景位置结束 | 结尾不停在黑屏、死亡界面或调试窗口 |

不要为了这条总览视频强行走完三条路线。只要展示“存在路线选择”，三路线的细节留给后续视频和作品集文字。
