# 《囚城营救》RAG 设定问答与来源引用设计

## 目标

制作一个本地运行的最低可用 RAG 原型。用户输入关于《囚城营救》的设定问题后，系统从精选游戏设定文档中检索相关片段，让 DeepSeek 根据片段回答，并显示答案来源。

第一版用于验证 RAG 的核心流程和作品集展示价值，不接入 Unity，不做冲突检查、支线生成或 AI NPC 集成。

## 核心使用场景

```text
问题：胆小守卫提供的真实路线在哪里？

回答：往前第二个口子跳下去有一条暗道，几乎无人镇守。

来源：
- docs/game_design.md
- docs/project-knowledge.md
```

成功标准：

- 能回答已有设定中的问题。
- 回答附带实际参与检索的来源文件。
- 找不到依据时明确回答“当前知识库没有足够信息”。
- 不根据模型常识擅自补全未确认设定。

## 第一版数据范围

索引文件：

- `docs/game_design.md`
- `docs/rag_setting.md`
- 后续新增的角色、阵营、地图、任务与剧情设定文档

不索引文件：

- `docs/progress.md`
- `docs/tech_decision.md`
- `docs/coding_rules.md`
- `HANDOVER.md`
- 录制指南、作品集介绍和开发实施文档

原因：第一版知识库只保存游戏世界与玩法事实，避免检索结果混入开发进度和技术说明。

## 技术方案

采用本地 Embedding 与 NumPy 向量检索：

```text
精选 Markdown 文档
-> 按标题和段落切分
-> 本地中文 Embedding 模型生成向量
-> 保存文本、来源和向量
-> 对用户问题生成向量
-> NumPy 计算余弦相似度
-> 返回最相关的 3 个片段
-> DeepSeek 根据片段回答
-> 输出回答与来源
```

第一版不使用 Chroma 或 FAISS。完成核心流程验证后，可以替换检索存储层，而不改变问答和来源引用接口。

## 组件边界

### 文档加载与切分

职责：

- 只读取允许索引的 Markdown 文件。
- 按 Markdown 标题和非空段落切分。
- 为每个片段保存来源文件和标题。
- 过滤过短、没有独立意义的片段。

片段结构：

```json
{
  "text": "真实安全路线：往前第二个口子跳下去有一条暗道，几乎无人镇守。",
  "source": "docs/game_design.md",
  "heading": "关卡结构"
}
```

### Embedding 与索引

职责：

- 使用本地中文 Embedding 模型生成片段向量。
- 将向量和片段元数据保存到本地。
- 重建索引时覆盖旧索引。

索引输出：

```text
rag_data/chunks.json
rag_data/embeddings.npy
```

### 检索

职责：

- 对问题生成向量。
- 使用余弦相似度排序。
- 返回最相关的 3 个片段及相似度。

第一版不设置复杂过滤条件。若最高相似度低于最低阈值，则视为知识库没有足够依据。

### 回答生成

职责：

- 只根据检索片段回答。
- 不补全片段中不存在的设定。
- 返回简短答案和去重后的来源列表。
- DeepSeek 调用失败时仍显示检索片段与来源。

返回结构：

```json
{
  "answer": "往前第二个口子跳下去有一条暗道，几乎无人镇守。",
  "sources": [
    "docs/game_design.md"
  ],
  "matches": [
    {
      "text": "真实安全路线：往前第二个口子跳下去有一条暗道，几乎无人镇守。",
      "source": "docs/game_design.md",
      "score": 0.82
    }
  ]
}
```

## 交互方式

第一版使用 Python 命令行：

```text
python build_index.py
python ask.py
```

`build_index.py` 负责创建索引。`ask.py` 连续接收问题并显示答案和来源，输入“退出”结束。

暂不制作网页或 Unity UI，避免提前扩展展示层。

## 异常处理

- 索引文件不存在：提示先运行 `build_index.py`。
- 精选设定文件不存在：显示具体文件路径并停止构建。
- 本地 Embedding 模型加载失败：显示清晰错误，不创建不完整索引。
- DeepSeek 调用失败：显示检索到的原文片段和来源。
- 检索依据不足：回答“当前知识库没有足够信息”，不让模型自由补全。

## 测试与验收

最低验收问题：

1. `胆小守卫提供的真实路线在哪里？`
   - 应检索到往前第二个口子跳下去的暗道。
2. `威胁胆小守卫可能得到什么情报？`
   - 应检索到往上走的虚假路线。
3. `玩家为什么要营救人质？`
   - 当前设定尚未确认，应回答知识库没有足够信息。

验收时检查：

- 回答是否符合文档事实。
- 来源文件是否正确。
- 未知问题是否拒绝编造。
- 重建索引后修改过的设定是否生效。
