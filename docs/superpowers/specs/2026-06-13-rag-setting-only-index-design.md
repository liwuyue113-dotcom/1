# RAG 专用设定索引范围设计

## 目标

让设定问答索引只包含经过确认和整理的专用设定文档，避免完整策划案中的通用玩法、作品集描述和待开发内容干扰检索排名。

## 方案

将 `rag/rag_config.py` 的 `SOURCE_FILES` 缩小为：

```python
SOURCE_FILES = [
    "docs/rag_setting.md",
]
```

`docs/game_design.md` 继续作为完整游戏策划文档维护，但不再直接进入设定问答索引。适合 RAG 回答的已确认事实应整理到 `docs/rag_setting.md` 或后续新增的专用角色、阵营、地图、任务和剧情设定文档。

## 原因

- `docs/rag_setting.md` 已按玩家问题拆成单义事实。
- `docs/game_design.md` 包含文档用途、核心玩法、评价指标和待开发构想，这些内容会干扰具体设定问答。
- 第一版只需要稳定、可解释的设定检索，不需要多索引或复杂章节过滤。

## 验收标准

- 自动化测试确认默认索引范围只包含 `docs/rag_setting.md`。
- 全部自动化测试通过。
- BGE 索引重建后保持 512 维。
- 五题 Top 3 检索全部通过。
- `主角为什么要营救人质？` 继续命中未确认设定并拒绝编造。

## 本阶段不做

- 不删除或重写 `docs/game_design.md`。
- 不新增多索引路由。
- 不增加按标题过滤规则。
- 不新增未确认游戏设定。
