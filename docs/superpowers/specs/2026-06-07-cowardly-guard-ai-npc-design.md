# 胆小守卫 AI NPC 设计

日期：2026-06-07
项目：《囚城营救》Unity 2D
目标场景：Level1

## 目标

在 Level1 中加入一个胆小守卫 AI NPC。守卫听见外面的砍杀声后擅自离岗并躲藏。玩家遇见他后，可以自由输入文字进行对话，并尝试获得后续三条路线的情报。

该功能需要形成可展示的玩法闭环：玩家语言影响守卫状态，守卫状态影响真假情报，真假情报影响限时营救中的路线选择。

## MVP 范围

第一版实现：

- 玩家使用自由文字输入与胆小守卫对话。
- DeepSeek 判断玩家话语属于安抚、威胁、质疑、询问路线或其他。
- Python 规则系统管理信任、恐惧和情报结果。
- 安抚可以逐渐获得真实安全路线。
- 威胁会增加恐惧，守卫可能提供虚假危险路线。
- 玩家可以质疑假情报并继续安抚，让守卫改口。
- 玩家首次触发守卫对话时，倒计时继续运行并且只增加一次 20 秒。
- 获得真实情报后，提示往前第二个口子跳下去存在一条几乎无人镇守的暗道。
- 获得假情报后，误导玩家往上走，但不改变敌人属性。
- DeepSeek 请求失败时，显示预设失败台词，不改变任何状态，并允许重试。
- 对话记忆只保留在当前一局游戏中，重新开始关卡后清空。

第一版暂不实现：

- 极端威胁直接获得真实路线。
- 小地图、路线高亮或场景箭头。
- RAG、向量数据库或多 Agent。
- 跨局永久记忆。

## 核心设计原则

采用“AI 理解语言，规则系统决定结果”的方案。

DeepSeek 负责：

- 判断玩家话语意图和强度。
- 根据守卫身份、情绪和本局历史生成自然回复。

Python 规则系统负责：

- 更新信任和恐惧。
- 决定是否透露情报。
- 决定情报是真实、虚假还是暂不透露。

Unity 负责：

- 显示对话和情绪状态。
- 控制倒计时奖励。
- 显示固定路线提示。
- 降低安全路线守卫血量。

AI 不允许编造地图路线，也不能直接操作 Unity 游戏对象。

## 角色与状态

胆小守卫听见战斗声后离开岗位，躲在城堡深处。他知道后续区域的巡逻安排，但误以为玩家可能杀死自己。

初始状态：

- 信任：0
- 恐惧：70
- 情绪显示：非常害怕
- 情报状态：未透露
- 额外时间奖励：未领取

玩家看不到具体信任和恐惧数值，只能看到大致情绪和守卫说话时的语气。

## 第一版规则

- 有效安抚：信任增加 10 到 20，恐惧降低 5 到 15。
- 威胁或辱骂：信任降低 10 到 20，恐惧增加 10 到 20。
- 询问路线时，信任大于等于 50 且信任大于等于恐惧：提供真实路线。
- 询问路线时，恐惧大于等于信任加 30：提供虚假路线。
- 其他情况：拒绝透露路线。
- 假情报不是最终结果；玩家后续达到真实路线条件时，守卫承认撒谎并更正情报。

后续进阶版可以使用预留的意图强度字段，实现“极端威胁立即获得真实路线，但不会获得守卫弱点”。

## 组件与职责

### Unity

- `AIGuardInteraction`：检测玩家交互、首次交互，并打开或关闭 AI 对话 UI。
- `AIGuardDialogueUI`：显示守卫回复、情绪、输入框、发送按钮和请求状态。
- `AIGuardApiClient`：通过 HTTP 请求 FastAPI，并处理成功、失败和超时。
- `RouteIntelManager`：显示真假路线文字；真实路线时降低指定守卫血量。
- `GameTimer`：增加公开加时方法；AI 守卫对话时不暂停倒计时。

现有普通剧情 NPC 继续使用 `NPCTalk` 和 NodeCanvas。

### Python / FastAPI

- 对话路由：接收玩家输入和本局会话标识。
- DeepSeek 客户端：判断意图并生成角色回复。
- 守卫状态服务：保存本局信任、恐惧、情报状态和对话历史。
- 规则引擎：根据意图和当前状态计算新状态与情报结果。

## 数据流

1. 玩家首次与守卫交互。
2. Unity 给倒计时增加 20 秒，并记录本局已经领取奖励。
3. 玩家输入文字并点击发送。
4. Unity 将文字和会话标识发送给 FastAPI。
5. FastAPI 调用 DeepSeek 判断意图。
6. 规则引擎更新信任、恐惧和情报状态。
7. DeepSeek 根据规则结果生成守卫台词。
8. FastAPI 返回固定 JSON。
9. Unity 显示台词和情绪，并执行固定路线效果。

## API 返回格式

```json
{
  "reply": "我……我确实知道一条巡逻较少的路。",
  "emotion": "稍微放松",
  "trust": 35,
  "fear": 45,
  "intent": "comfort",
  "intensity": "normal",
  "intel_result": "none"
}
```

`intel_result` 只允许：

- `none`
- `true_route`
- `false_route`

Unity 不向玩家直接显示 `trust` 和 `fear`，它们用于规则执行和开发调试。

## 路线效果

真实路线：

- 显示“往前第二个口子跳下去有一条暗道，几乎无人镇守”。
- 指向几乎无人镇守的暗道。
- 降低该路线指定守卫的最大生命值和当前生命值。

虚假路线：

- 显示“往上走最安全”的误导提示。
- 指向上方的危险路线。
- 不额外增强敌人，也不生成增援。

## 异常处理

- API 失败或超时：显示“我……我现在脑子很乱，你再说一遍。”
- API 失败时不改变信任、恐惧或情报状态。
- 发送请求期间禁用发送按钮，避免重复请求。
- DeepSeek 返回格式错误时，由后端拦截并使用失败响应。

## 测试与完成标准

规则测试至少覆盖：

- 多次安抚后获得真实路线。
- 普通威胁后获得假路线。
- 假路线之后继续安抚，最终改为真实路线。
- 非询问路线的普通对话不会意外透露路线。
- API 失败不改变状态。

Unity 验收标准：

- Level1 中可以自由输入文字与守卫对话。
- 首次交互只增加一次 20 秒，且对话期间倒计时继续。
- UI 显示守卫回复和大致情绪。
- 真假情报显示不同的固定路线提示。
- 真实情报会降低安全路线指定守卫血量。
- API 失败时允许重试。

## 作品集展示重点

展示视频应呈现两次不同选择：

- 安抚守卫，获得真实安全路线与守卫弱点。
- 威胁守卫，被虚假路线误导。

架构说明需要突出：AI 负责理解玩家语言，规则系统保证玩法公平，Unity 负责执行游戏机制。
